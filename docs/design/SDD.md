> 版本：v0.1 / 建立日期：2026-07-04

## 1. 系統概述

個人使用的台股現股買賣建議輔助系統。由 Claude Code 於每日 08:30 / 12:30 / 21:30 三個時間點本機排程執行分析，蒐集公開市場資料，產出買賣建議報告，透過 GitHub Pages 靜態網頁供使用者瀏覽。系統邊界與核心原則見 `docs/requirements/需求.md`。

## 6. 核心規則（衍生自 `docs/requirements/需求.md`，需求變更時須同步此節）

`docs/requirements/需求.md` 是持續演進的需求來源（已改版多次），下游程式（`CLAUDE.md`、`job/blueprint.md`、`job/prompts/*.md`）一律引用本節，不直接引用需求文件的章節編號，避免需求文件改版、章節重排時下游跟著默默跑掉。

### 6.1 核心原則（鐵律）

1. 不碰選擇權，只做現股部位判斷。
2. 反散戶心理：融資暴增＝浮額過熱（不追高）；融資斷頭＝乾淨籌碼（價值獵殺點）。
3. 金字塔定網：只給限價低接建議，不給市價單追價。
4. 只給建議，不執行交易：不連接券商 API，不知道使用者真實庫存與資金水位，所有金額/張數建議僅供參考。

### 6.2 資料正確性原則

- 同一項關鍵數據（股價、融資融券、法人買賣超）盡量從多個公開來源交叉比對。
- **禁止在抓不到資料時用推測值或記憶中的舊數字填充**。任一關鍵數據抓不到或來源互相矛盾時，該欄位必須明確標示「資料未取得／來源不一致」。
- **禁止查詢未篩選代號的全市場總表**（如 TWSE `MI_MARGN`、`T86` 未指定代號的整份清單）：表格過大時 WebFetch 摘要模型有機率編造畫面中實際不存在的數字（2026-07-04 手動驗證時實測發生，見 ADR-003）。一律改用已篩選單一代號的個股頁面（如 Yahoo 奇摩股市個股融資融券／法人買賣超頁）查詢，且查詢時需明確要求「找不到就回報找不到，不可假設或推算」。

### 6.3 報告欄位規範

每份報告固定包含以下欄位區塊：

- **A. 總經與市場底座**：費城半導體指數漲跌幅、WTI 原油價格、加權指數點位與成交量。
- **B. 關注個股籌碼表**：融資/融券餘額單日增減、三大法人買賣超張數（外資／投信／自營商）；21:30 為當日正式數據，其餘時段沿用前一交易日數據。
- **C. 建議動作**：標籤【金字塔低接／觀望看戲／高位停利變現】三選一；限價區間（一律限價，不給市價單追價建議）；判斷理據。
- **D. 資金安全警語與回補提示**：固定印出 `[CASH_WARNING]` 警語；若先前建議「高位停利變現」，需提示「該檔待回補」，直到重新建議「低接」為止。

### 6.4 三時段任務差異（本時段定位）

- **08:30 PRE（盤前分析）**：抓總經指標、沿用前一交易日籌碼結論、計算金字塔低接與高位停利區間。
- **12:30 MID（盤中複核）**：複核 08:30 建議是否已觸及／仍成立，**非追即時大單**（公開頁面報價有 delay），不做全新分析。
- **21:30 POST（盤後定錨）**：結算當日正式公告的籌碼數據、處理使用者丟入的連結、規劃明日操盤方向。
