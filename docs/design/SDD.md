# 系統設計說明書（SDD）— BLACKJ-STOCK-POINT-GUARD

> 版本：v0.1 / 建立日期：2026-07-04

## 1. 系統概述

個人使用的台股現股買賣建議輔助系統。由 Claude Code 於每日 08:00 / 12:30 / 21:30 三個時間點本機排程執行分析，蒐集公開市場資料，產出買賣建議報告，透過 GitHub Pages 靜態網頁供使用者瀏覽。系統邊界與核心原則見 `docs/requirements/需求.md`。

## 6. 核心規則（衍生自 `docs/requirements/需求.md`，需求變更時須同步此節）

`docs/requirements/需求.md` 是持續演進的需求來源（已改版多次），下游程式（`CLAUDE.md`、`job/blueprint.md`、`job/prompts/*.md`）一律引用本節，不直接引用需求文件的章節編號，避免需求文件改版、章節重排時下游跟著默默跑掉。

### 6.1 核心原則（鐵律）

1. 不碰選擇權，只做現股部位判斷。
2. 反散戶心理：融資暴增＝浮額過熱（不追高）；融資斷頭＝乾淨籌碼（價值獵殺點）。
3. 金字塔定網：只給限價低接建議，不給市價單追價。
4. 只給建議，不執行交易：不連接券商 API，不知道使用者真實庫存與資金水位，所有金額/張數建議僅供參考。

### 6.2 資料正確性原則

- 同一項關鍵數據（股價、融資融券、法人買賣超）盡量從多個公開來源交叉比對。
- **禁止在抓不到資料時用推測值或記憶中的舊數字填充**。任一關鍵數據抓不到或來源互相矛盾時，該欄位必須明確標示「資料未取得／來源不一致」。
- **禁止查詢未篩選代號的全市場總表**（如 TWSE `MI_MARGN`、`T86` 未指定代號的整份清單）：表格過大時 WebFetch 摘要模型有機率編造畫面中實際不存在的數字（2026-07-04 手動驗證時實測發生，見 ADR-003）。一律改用已篩選單一代號的個股頁面（如 Yahoo 奇摩股市個股融資融券／法人買賣超頁）查詢，且查詢時需明確要求「找不到就回報找不到，不可假設或推算」。

### 6.3 報告欄位規範

每份報告固定包含以下欄位區塊：

- **A. 總經與市場底座**：費城半導體指數漲跌幅、WTI 原油價格、加權指數點位與成交量；PRE 時段另加日經 225 指數、韓國 KOSPI 指數即時漲跌幅（日韓股市開市時間較台股早 1 小時，可作台股開盤前區域市場情緒參考，見 6.4）。
- **B. 關注個股籌碼表**：融資/融券餘額單日增減、三大法人買賣超張數（外資／投信／自營商）；21:30 為當日正式數據，其餘時段沿用前一交易日數據。
- **C. 建議動作**：標籤【金字塔低接／觀望看戲／高位停利變現】三選一；限價區間（一律限價，不給市價單追價建議）；判斷理據。
- **D. 資金安全警語與回補提示**：固定印出 `[CASH_WARNING]` 警語；若先前建議「高位停利變現」，需提示「該檔待回補」，直到重新建議「低接」為止。
- **延續數據表**（見 6.5）：每份報告固定附上一張「代號／名稱／收盤價／第一批限價」小表格，供前台重點摘要 lightbox 讀取；本次有重新查價/計算就用最新值，沒有就延續 `state.json` 的快照值並標明為延續，不得省略此表。

### 6.5 延續數據表與 `reports/state.json` 擴充欄位

為了讓前台「重點摘要」在 12:30／21:30 報告也能顯示股價與限價區間（不只 08:30 才有），`reports/state.json` 每檔股票新增兩個欄位：

- `last_price`：最近一次查證到的收盤價（字串，如 `"110.00"`）
- `limit_range`：金字塔限價低接第一批價格區間（字串，如 `"107 - 108"`；若目前建議為「高位停利變現」可留空或維持前次數值供參考，不代表現在建議低接）

**規則**：
1. **08:00 PRE**：計算完限價區間後，把 `last_price`／`limit_range` 一併寫回 `state.json`。
2. **12:30 MID／21:30 POST**：無論當次是否重新查價，報告內文都必須包含「延續數據表」（代號／名稱／收盤價／第一批限價），資料來源可以是本次重新查到的價格，也可以是延續 `state.json` 的既有快照值——**延續使用快照值不算違反 6.2 資料正確性原則**（不是憑空推測，是明確標示「沿用先前已查證的數字」），但報告文字需清楚寫明「延續 08:00 快照」或「本次重新查證」，不得含糊。若本次有重新查到價格，同步更新 `state.json` 的 `last_price`（`limit_range` 除非重新分析，否則維持不變）。
3. 此表格式固定為 4 欄表格，表頭需包含「收盤價」與「限價」字樣，供 `web/build.py` 辨識擷取（見 ADR-005）。

### 6.4 三時段任務差異（本時段定位）

- **08:00 PRE（盤前分析）**：抓總經指標（含日經 225／韓國 KOSPI 早盤動向，見 6.3）、沿用前一交易日籌碼結論、計算金字塔低接與高位停利區間。
- **12:30 MID（盤中複核）**：複核 08:00 建議是否已觸及／仍成立，**非追即時大單**（公開頁面報價有 delay），不做全新分析。
- **21:30 POST（盤後定錨）**：結算當日正式公告的籌碼數據、處理使用者丟入的連結、規劃明日操盤方向。
