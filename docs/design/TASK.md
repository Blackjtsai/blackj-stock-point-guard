## Layer 4：端對端整合測試

**UC 範圍**：全部 UC

**目標**：完整跑過一天三個時間點的排程，確認從資料蒐集到前台顯示全鏈路正確，且三種 error path 都符合預期行為（不拋錯、不捏造、不重試）。

**驗收條件**：

| 項目 | 成功路徑 | 失敗路徑 |
|---|---|---|
| 三次排程實跑 | 08:30/12:30/21:30 皆自動觸發並產出報告 | 任一次未觸發 → 檢查 launchd 設定 |
| 前台顯示 | 三份報告皆可在 GitHub Pages 看到 | 顯示不到 → 檢查部署路徑與 git push |
| 資料抓不到 | 報告明確標示「資料未取得」 | 報告出現看似合理但無法查證的數字 → 視為缺陷，需修正 prompt |
| 機器未開機/未登入 | 該次排程無報告產生，無錯誤訊息干擾使用者 | 出現錯誤通知 → 違反「不需要通知」原則，需修正 |
| 訂閱額度不足 | 該次排程失敗即結束 | 自動重試導致額度持續消耗 → 需修正 |

**依賴**：Layer 1、2、3 全部 `[x]`
