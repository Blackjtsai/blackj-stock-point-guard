# ADR-006：真實成本價護盾續抱規則與除權除息官方查證來源

> 狀態：已採納
> 日期：2026-07-06
> 決策者：黑暗傑客（Eason Tsai）／Claude Code

## 背景

使用者對 2026-07-06 08:00 PRE 報告提出三點優化指示，逐項查核後：

1. **英業達(2356) 除息日 2026-07-15、每股 NT$2.00**：查核台灣證交所官方除權除息預告表 API（`https://www.twse.com.tw/rwd/zh/exRight/TWT48U?response=json`）後確認此數字**正確**，並非幻覺。但排程流程中原本沒有任何步驟去查證這類資料，這次只是剛好蒙對——這是流程缺口，不是這次的結果錯誤。
2. **大聯大(3702) 法人數據因 API 誤標「上緯投控」而集體顯示未取得**：查核歷史報告後確認這件事發生在 2026-07-04，且當天 21:30 POST 報告已改查 Yahoo 個股頁面修復，2026-07-06 的報告本身沒有這個問題。備援機制目前是臨時手動處理，沒有寫進 blueprint 變成固定流程。
3. **建準(2421) 現價逼近原設定的高位停利參考區間（156~161）就建議部分停利，被認為是短線散戶思維**：使用者實際持股成本遠低於現價（見下方成本快照），系統原本的三態決策邏輯完全不知道這件事（`docs/design/SDD.md` 第 6.1 條核心原則第 4 項明訂「不知道使用者真實庫存」，`CLAUDE.md` 關鍵業務約束亦有對應描述），導致同一套限價/停利區間邏輯無差別套用在「純觀察名單」與「已有大幅浮盈的真實持股」上。

## 選項

**成本資料存放位置**：

| 選項 | 優點 | 缺點 |
|---|---|---|
| A. 寫進 `job/watchlist.json` 或 `reports/state.json`，隨排程一起 commit/push | 排程讀取路徑最短，不需額外檔案 | 這個 repo 是 **Public**，會把真實持股成本與（可推算的）未實現損益狀況公開；比目前「股票代號公開但無成本資訊」曝露程度高出一截，使用者未事先同意到這個程度 |
| B. 新增本機專用檔案 `job/holdings.local.json`，加進 `.gitignore`，排程唯讀 | 真實成本價與持股規模完全不進版控，repo 曝露程度不變；排程仍可讀取套用邏輯（Read 不受 `--allowedTools` 網域限制） | 換到常駐機器時（見既有 TODO「遷移到常駐機器」待辦）需要手動另外同步這份本機檔案，不會隨 `git pull` 自動帶過去 |

使用者於對話中明確選擇 **選項 B**（存在本機、不進版控）。

**是否修改核心決策邏輯**：

| 選項 | 優點 | 缺點 |
|---|---|---|
| A. 維持現狀，只調整語氣措辭 | 不動架構 | 不解決根本問題：系統仍無法區分「應咬住大波段的真倉位」與「單純限價觀察名單」 |
| B. 新增「安全墊 ≥5% 時預設不給高位停利變現」的護盾規則，僅對 `holdings.local.json` 有記錄的代號生效 | 直接對應使用者訴求；不影響純觀察名單（無真實持股的 7 檔）原有限價邏輯；有「大盤系統性崩盤」明確覆蓋條件，不會變成無論如何都不許停利的僵化規則 | 需要定義「系統性崩盤」的量化門檻，門檻訂得不好可能過鬆或過嚴 |

選擇 **選項 B**。

## 決策

1. **成本資料**：新增 `job/holdings.local.json`（本機專用，`.gitignore` 排除），記錄真實持股代號、股數、`cost_basis`；使用者手動維護，排程只讀不寫。
2. **護盾續抱規則**（`docs/design/SDD.md` 第 6.6 節）：`holdings.local.json` 有記錄的代號，安全墊 `(現價-cost_basis)/cost_basis ≥ 5%` 時，C 段建議動作預設不給「高位停利變現」，維持「觀望看戲」並註明護盾理由；例外為大盤系統性崩盤（台灣加權指數單日跌幅 ≥3%，或 SOX 單日重挫 ≥5% 且台股同步顯著走弱）時可正常評估停利。安全墊 <5% 或代號不在此檔案中，不受影響，維持原本三態邏輯。
3. **除權除息查證**：`docs/design/SDD.md` 第 6.2 節新增規則，任何除權除息日期／股利數字必須先用 TWSE 官方 `TWT48U` API 查證，禁止憑訓練知識或沿用舊報告數字斷言（即使這次數字剛好正確，流程本身要補上查證步驟）。
4. **法人數據備援順序**：明訂為 Yahoo 個股頁 → `finance.yahoo.com` 個股頁 → 標示「資料未取得」，任何情況下不得回退到 `T86`／`TWT44U` 全市場總表（見 ADR-003 追記，實測證實這兩支端點不支援單一代號篩選，回退等於重新引入已被禁止的風險）。

後果：
- `docs/design/SDD.md`（v0.8 → v0.9）、`job/blueprint.md`（v0.8 → v0.9）、`job/prompts/{PRE,MID,POST}.md` 同步更新。
- **代價**：多一份本機檔案需要手動維護，且遷移到常駐機器時需要额外同步（已記錄於 `docs/TODO.md` 待辦區）；「系統性崩盤」門檻是主觀訂定的量化值，未來若證實不合用需再修正並補 ADR。
- 若使用者之後想讓這份成本資料也能在其他機器上使用（例如常駐機器排程），需要重新決定是否要有一個不進 Public repo 的私有同步機制（比 `.gitignore` 本機檔案更進一步），目前不在本次決策範圍內。
