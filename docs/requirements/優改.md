## 2. 實際產出的 Markdown 報告欄位規範

原第 7 節之輸出格式，不論哪個時段的 Markdown 報告，其實際產出的數據表格與內文必須硬性包含以下具體股票性質欄位：

### A. 總經與市場底座區塊

* **國際催化劑**：費城半導體指數最新漲跌幅（%）、WTI 原油價格（美金/桶）。
* **大盤心理測謊**：當前/今日加權指數點位、預估/實際成交量，用以判斷市場處於「散戶 FOMO 追高」或「非理性恐慌踩踏」。

### B. 關注個股籌碼測謊表（Watchlist Table）

針對 4 檔核心大軍（3702、2356、2609、2421），必須動態填入：

* **融資/融券餘額單日增減**（21:30 完整更新，其餘時段讀取前日餘額）。
* **三大法人買賣超張數**（外資、投信、自營商）。

### C. 黑暗傑客 ── 終極操盤指令欄位（決策官核心）

* **建議動作**：明確給出【金字塔低接 / 觀望看戲 / 高位停利變現】標籤。
* **冷血定網價位**：**嚴禁提供市價單追價建議**，一律提供精算後的現股限價（Limit Order）區間。
* **判斷理據**：結合散戶心理學（如融資斷頭、高檔浮額過熱）之冷酷文字點評。

### D. 資金防禦與回補警告提示（核心安全墊）

* **資金流動性盲區警告**：由於系統不串接券商 API，報告最上方或最下方必須硬性印出以下警語：
> `[CASH_WARNING]`：「本建議未包含大後方真實流動性計算。請主理人（黑暗傑客）人工核對 T+2 日真實交割扣款金額，確保銀行活存防禦墊水位安全後，方可執行下單。」


* **回補進度追蹤**：提示個股（如陽明、建準）之前高位套現後，目前尚需回補之預估張數與所需準備金提示。
