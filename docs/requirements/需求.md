# BLACKJ-STOCK-POINT-GUARD 需求文件

**中文名稱**：黑暗傑客股市控球後衛
**系統代碼**：BJSPG
**版本**：v2 ／ 建立日期：2026-07-04

---

## 1. 專案目標

由 Claude Code 在每日固定時間點自動執行分析任務，產出台股現股（不碰選擇權）的**買賣建議**，寫入本機專案資料夾，並透過一個簡易前台讓使用者依日期瀏覽當天產出的報告。

系統不連接券商、不觸碰真實帳戶、不自動下單。所有決策仍由使用者本人（老大）人工執行。

---

## 2. 核心原則（鐵律）

1. **不碰選擇權**，只做現股部位判斷。
2. **反散戶心理**：融資暴增視為浮額過熱（不追高）；融資斷頭視為乾淨籌碼（價值獵殺點）。
3. **金字塔定網**：只給限價低接建議，不建議市價單追價。
4. **只給建議，不執行交易**：系統不連接任何券商 API，不知道使用者真實庫存與資金水位，所有金額/張數建議僅供參考。

---

## 3. 系統邊界（Scope & Non-Goals）

| 項目             | 決定                                                                                    |
| ---------------- | --------------------------------------------------------------------------------------- |
| 券商系統串接     | ❌ 不做。不追蹤真實庫存、不算真實 T+2 交割款                                             |
| 自動下單         | ❌ 不做。只出建議                                                                        |
| Google Docs 公文 | ❌ 不用。改為寫入專案資料夾內的檔案                                                      |
| 執行引擎         | ✅ 直接由 Claude Code 執行分析與寫檔，不另外呼叫 Gemini API                              |
| 資料取得方式     | ✅ Claude Code 於排程觸發時，透過 WebSearch / WebFetch 即時上網查詢公開資料（見第 6 節） |
| 前台             | ✅ 需要一個可依日期瀏覽歷史報告的簡易網頁介面                                            |

---

## 4. 每日排程時間軸

配合台股實際交易時段（09:00–13:30）與公開資料公告進度，時間點微調如下：

| 時間  | 報告代碼       | 任務     | 核心工作內容                                                                                                                                                       |
| ----- | -------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 08:30 | `0830_PRE.md`  | 盤前分析 | 1. 抓費半指數、WTI 原油最新收盤數據 2. 讀取前一交易日籌碼結論（融資融券、三大法人）3. 計算關注股「金字塔限價低接區間」與「高位停利區間」                           |
| 12:30 | `1230_MID.md`  | 盤中複核 | **非追即時大單**（公開資料有 delay，不適合抓即時大單）。1. 抓大盤與關注股當下高低點 2. 推測 08:30 限價建議是否已成交 3. 若大盤盤中急跌，評估是否撤單觀望或維持原判 |
| 21:30 | `2130_POST.md` | 盤後定錨 | 1. 結算當日正式公告的融資融券餘額增減與三大法人買賣超 2. 拆解使用者當日輸入的 YouTube/新聞連結 3. 規劃明日操盤方向                                                 |

**執行方式：本機排程**。使用 macOS `launchd`（或 `cron`）在上述時間點呼叫 `claude` CLI，以使用者自己的 Claude 訂閱帳號（非另接 API 計費）headless 執行分析流程。**僅平日（週一至週五）觸發，週六日台股休市不執行**。前提：執行當下電腦需開機、登入、有網路連線；若當下用量額度不足或機器未開機，該次排程直接跳過，不做額外重試或通知（與第 8 節「不需要通知」一致）。

---

## 5. 角色設計（分析流程中的邏輯分工）

**決定：單一流程**。不做成獨立 subagent，而是同一次 Claude Code 執行內，依序扮演以下邏輯角色，最終彙整成一份報告：

1. **籌碼偵察兵**：查融資融券餘額增減、三大法人買賣超，判斷浮額過熱 / 乾淨籌碼。
2. **總經體檢員**：查 WTI 原油、費城半導體指數等總經指標，確認基本面底座是否穩固。
3. **產業分析師**：針對關注清單標的（見第 9 節），視需要拆解護城河；使用者可隨時丟入 YouTube 連結或新聞連結，要求針對特定個股做主力進出貨階段判斷。
4. **彙整決策官**：整合以上輸出，產出「當日買賣建議報告」，包含建議標的、建議動作（加碼低接 / 觀望 / 高檔減碼）、建議價位區間、判斷理由。

---

## 6. 資料來源與取得方式

執行期由 Claude Code 直接使用 **WebSearch / WebFetch** 工具即時瀏覽公開網頁取得資料，不需另外申請券商或付費行情 API。適用資料：

- 證交所公開資訊（融資融券餘額、三大法人買賣超）
- Yahoo 奇摩股市、財經新聞網站等公開即時報價頁面
- WTI 原油價格、費城半導體指數
- 使用者提供的 YouTube 連結 / 新聞連結

**資料正確性原則**：
- 同一項關鍵數據（股價、融資融券、法人買賣超）盡量從多個公開來源交叉比對，取一致或較新的數據，避免單一來源錯誤或延遲誤導判斷。
- **禁止在抓不到資料時用推測值或記憶中的舊數字填充**。任一關鍵數據確實抓不到或來源互相矛盾時，報告該欄位必須明確標示「資料未取得／來源不一致，請自行查證」，不得假裝有完整數據。

**已知限制（已確認可接受）**：
- 公開頁面的盤中報價通常有 delay（非付費即時 tick），因此 12:30 已改為「複核」而非追蹤即時大單（見第 4 節）。
- 沒有券商 API，無法得知使用者實際庫存與現金水位，因此不會有「精算交割款」這類功能，僅能給通用建議。

## 7. 輸出格式與儲存位置

- 不寫 Google Docs，改寫入專案資料夾，例如 `reports/2026-07-04/0830_PRE.md`、`reports/2026-07-04/1230_MID.md`、`reports/2026-07-04/2130_POST.md`。
- 格式：Markdown，每份報告必須包含以下欄位區塊：

**A. 總經與市場底座**
- 費城半導體指數最新漲跌幅、WTI 原油價格
- 加權指數點位、成交量，判斷市場處於「追高過熱」或「恐慌拋售」

**B. 關注個股籌碼表**（4 檔核心關注股）
- 融資/融券餘額單日增減（21:30 為當日正式數據，其餘時段沿用前一交易日數據）
- 三大法人買賣超張數（外資／投信／自營商）

**C. 建議動作**
- 標籤：【金字塔低接 / 觀望看戲 / 高位停利變現】三選一
- 限價區間：**一律提供限價（Limit Order）建議，不提供市價單追價建議**
- 判斷理據：結合融資增減、法人籌碼、總經指標的文字說明

**D. 資金安全警語與回補提示**
- 每份報告固定印出警語：
  > `[CASH_WARNING]`：本建議未包含真實庫存與資金流動性計算，請自行核對 T+2 交割扣款金額與銀行活存水位後再執行下單。
- 回補提示：若系統先前曾對某檔標的建議「高位停利變現」，之後應在報告中提示「該檔待回補」及參考回補價位，直到系統記錄該檔已重新建議「低接」為止。此提示僅根據**系統自身歷史建議記錄**推算，不代表使用者真實持股狀態。

**狀態記錄機制**：為支援上述「回補提示」，系統需在專案資料夾維護一份輕量歷史狀態檔（例如 `reports/state.json`），記錄每檔關注股最近一次的建議動作，供下次執行時比對。

## 8. 前台（Dashboard）需求

- **技術型態：純靜態 HTML + GitHub Pages**。每次報告產出後，同步更新靜態頁面資料，`git push` 到 GitHub，並透過 GitHub Pages 部署，讓使用者可以直接用手機/任何裝置上網瀏覽，不需要跑本機 server。
- **RWD**：頁面需支援響應式設計，手機與桌機瀏覽都要能正常閱讀。
- 使用者依日期瀏覽當天已產出的報告（08:30 / 12:30 / 21:30 三份），採簡單列表（最新在最上方），不做日曆選擇器。
- 不需要新報告產出通知，使用者自行找時間上網查看。
- GitHub repo 為 **Public**。
